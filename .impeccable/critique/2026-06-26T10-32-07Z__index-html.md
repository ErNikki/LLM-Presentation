## Priority Issues

- **[P2] Help is recall-dependent.** Shortcuts (←/→, Shift+0, F) live only in a kbd-hint that fades; no persistent legend or `?` overlay. Heuristic #10 = 2.
  - **Why it matters:** A presenter who forgets Shift+0 mid-talk has no recovery path; first-time viewers never learn the controls.
  - **Fix:** A dismissible `?`-toggle help overlay listing shortcuts, or a persistent low-key legend.
  - **Suggested command:** `/impeccable onboard` (or `/impeccable harden`)

- **[P2] Theming debt — brand colors hardcoded in diagram CSS.** 115 `rgba()` + 31 hex literals re-express brand hues instead of tokens.
  - **Why it matters:** The diagrams can't be re-themed by changing tokens; every hue tweak is a manual sweep.
  - **Fix:** Replace recurring brand-hue literals with `color-mix(in oklch, var(--accent) N%, …)` or alpha tokens.
  - **Suggested command:** `/impeccable extract`

- **[P2] Overview cards lack full keyboard support.** Focus styling is in place, but `.ov-card` (JS-generated divs) need `tabindex` + Enter handling to be reachable/activatable by keyboard.
  - **Why it matters:** The Shift+0 index is a key navigation aid; keyboard users can open it but can't tab/activate the thumbnails.
  - **Fix:** Add `tabindex="0"` + keydown(Enter/Space) in the overview render.
  - **Suggested command:** `/impeccable harden`

- **[P3] Undocumented gold reward hue (×4).** `#caa11e` / `#8a6d10` / reward yellow in the transformer/RL figure, outside DESIGN.md.
  - **Fix:** Add "Reward Gold" to DESIGN.md + sidecar.
  - **Suggested command:** `/impeccable document`
