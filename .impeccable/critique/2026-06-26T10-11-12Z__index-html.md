## Anti-Patterns Verdict

**Does this look AI-generated? Partly — the chrome does, the substance doesn't.**

**LLM assessment.** The hand-built inline SVG diagrams are genuinely custom and good — they pull the deck well above template territory and serve the "diagram is the argument" principle. But the *chrome* carries a stack of 2024–2026 AI-deck tells: the purple→teal gradient applied as **gradient text** in 16 places, soft radial background washes, a glassmorphic HUD/nav, and a tracked-uppercase numbered eyebrow on nearly every slide. Someone scanning the cover and a couple of content slides could reasonably say "AI made the styling," even though the diagrams say otherwise.

**Deterministic scan** (`detect.mjs`, exit 2, 8 findings):
- **numbered-section-markers** (advisory) — sequence 01–07 used as section labels.
- **overused-font** + **single-font** (warning) — Inter. *Partial false positive on "single font": JetBrains Mono is used for labels/counters/code, so the pairing exists; the "overused Inter" point stands.*
- **em-dash-overuse** (warning) — 20 em-dashes. *Low concern: this is Italian prose, where the dash is idiomatic; not a strong tell here.*
- **design-system-color** ×4 (advisory) — gold/amber (`#caa11e`, `#8a6d10`, `#0a9d78`, `rgba(245,193,66,…)`) in the transformer/RL diagram, outside DESIGN.md. *Legitimate: it's a deliberate 4th signal (reward/"+1"), but it's undocumented drift.*

**Visual overlays.** No browser-automation tool was available in this run, so no live in-page overlay was injected. Findings above are from the deterministic CLI scan + manual source review (fallback signal).

## Overall Impression

A well-engineered, genuinely educational deck whose *content layer* (custom SVG mechanisms, keyboard-first navigation, full reduced-motion support) is strong, dragged down by a *styling layer* that reaches for the exact AI-slop reflexes your PRODUCT.md rejects. The single biggest opportunity: strip the gradient text and the reflexive numbered eyebrows, and the deck instantly reads as "precise, lucid, unshowy" instead of "generated."

## What's Working

1. **The diagrams.** Hand-built SVG with a disciplined two-voice fill language (violet/teal). This is the hard part and it's done well — real custom work, not stock illustration.
2. **System status & navigation.** Progress bar + section label + `n/total` counter + a Shift+0 overview grid. A presenter and audience always know position and progress. Keyboard-first with a mouse fallback.
3. **Motion discipline at the floor.** A global `prefers-reduced-motion` block kills all animation and falls back to a 0.2s crossfade. Accessibility basic, handled.

## Priority Issues

- **[P1] Gradient text, 16 instances.** `.grad` = `background-clip:text` over the violet→teal gradient ("degli LLM" on the cover, inline emphasis spans throughout).
  - **Why it matters:** It's an absolute anti-pattern and the deck's loudest AI tell; it directly contradicts your "unshowy / earn every flourish" principle. On a projector, clipped gradient text also loses contrast and edge crispness versus solid ink.
  - **Fix:** Replace with solid color (ink for headings, or a single accent where emphasis is meaningful). Carry emphasis with weight/size, not gradient fill.
  - **Suggested command:** `/impeccable quieter` (or `/impeccable typeset`)

- **[P1] `text-faint` (#898ea3) fails contrast on a projected deck.** ~2.8:1 on the #f6f7fb page — below the 4.5:1 floor for normal text. Used on footnotes, examples (`.eg`), the cover meta, and the kbd-hint.
  - **Why it matters:** Your own design principle is "projection-legible by default — reads from the back row." This is the exact muted-gray-on-near-white failure that breaks in a bright room.
  - **Fix:** Darken the faint ramp to ~#5f6478 (≈4.6:1) for any real copy; reserve #898ea3 only for genuinely decorative micro-text.
  - **Suggested command:** `/impeccable colorize` (or fold into `/impeccable polish`)

- **[P2] Numbered tracked eyebrows on nearly every slide (12×).** `01 · Introduzione`, `01 · Introduzione`, … applied by reflex, not just where order matters.
  - **Why it matters:** Eyebrow + numbered-scaffolding is a doubled AI tell. The "Ciclo di una Richiesta" section genuinely *is* an ordered sequence (the numbers earn their place there); repeating `01 · Introduzione` on every intro slide is the reflex.
  - **Fix:** Keep numbers only in the lifecycle sequence. Drop them elsewhere; let the HUD section label carry context, or use a plain named kicker.
  - **Suggested command:** `/impeccable typeset` (or `/impeccable quieter`)

- **[P2] Missing visible focus states on controls.** Only `.demo-input:focus` exists. `.btn`, `.nav-arrow`, and the overview thumbnails have no `:focus-visible` ring.
  - **Why it matters:** Document-level key handlers cover slide nav, but a keyboard/AT user tabbing to the on-screen buttons gets no visible focus. Accessibility basic.
  - **Fix:** Add a consistent `:focus-visible` ring (2px violet, 2–3px offset) to all interactive controls.
  - **Suggested command:** `/impeccable audit` (or `/impeccable harden`)

- **[P3] Undocumented 4th diagram hue (gold/amber).** The transformer/RL figure uses `#caa11e` / `#8a6d10` / reward yellow, outside DESIGN.md.
  - **Why it matters:** It's a *good* use (a 4th signal = reward), but per your Two-Voice Rule a 4th hue must be a documented, deliberate meaning — right now it's drift.
  - **Fix:** Add "Reward Gold" to DESIGN.md + the sidecar tonal ramps as the explicit 4th signal, or recolor to an existing role.
  - **Suggested command:** `/impeccable document` (or `/impeccable colorize`)

## Persona Red Flags

**Sam (Accessibility-Dependent).** `text-faint` body/caption copy at ~2.8:1 fails WCAG AA — unreadable for low-vision users and on a projector. No visible focus ring on `.btn` / `.nav-arrow` / overview thumbnails when tabbing. Positive: reduced-motion fully respected, images carry alt text.

**Alex (Power Presenter, driving the deck).** Well served — ←/→, Shift+0 index, F fullscreen, instant jump via overview. Red flag: shortcuts live only in a kbd-hint that fades; no persistent legend or `?` help overlay, so a presenter who forgets Shift+0 has no recall path mid-talk.

**Marco (back-row attendee — project persona, technical colleague on a projector).** Gradient-clipped headings and #898ea3 captions wash out from across a room under projector light. The custom diagrams read well; the supporting micro-text and the cover title's gradient do not.

## Minor Observations

- Cover title clamp max is **7rem**, above the ~6rem (96px) display ceiling — it's shouting slightly. Consider ≤6rem.
- Glass HUD/nav (`backdrop-filter`) is borderline-decorative but defensible as functional, unobtrusive chrome — not a priority.
- "Overused Inter" is acceptable here: the Inter + JetBrains Mono pairing on a real contrast axis (sans vs mono) gives it identity. No action needed.

## Questions to Consider

- What would the cover look like with the title in solid ink and the *only* color being one deliberate accent word — does it get more confident, not less?
- The numbered markers: which sections are genuinely a sequence the audience must track in order, and which just inherited the pattern?
- If this is projected in a bright room, is anything currently set in #898ea3 actually important enough to be readable — and if not, should it exist at all?
